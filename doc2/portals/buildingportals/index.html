Choosing and Building Portal Websites

This page gives very simple instructions on setting up portal websites that serve as the introductory front-end to your Oracc corpus.

The options

There are five different ways to create a portal for your Oracc corpus, depending on your particular needs:

  1. If you would like a site that looks like K&P [http://oracc.museum.upenn.edu/saao/knpp/], GKAB [http://oracc.museum.upenn.edu/cams/gkab/], and AEB [http://www.ucl.ac.uk/sargon/], you can use ESP to build your portal site in situ in your project's 00web/ directory. This is our recommended option.

  2. You can ask your project liaison to create a separate project on Oracc to host your portal site, and build it using ESP. This is particularly useful if the project and corpus have different remits, managers, funders, etc. In this case, your html files will live in your portal project's (not your corpus project's) 00web directory on Oracc, with subdirectories as dictated by the structure of the website.

  3. You can create and maintain a portal site on a different server and have your Oracc project link to it by editing the configuration file. This can be helpful if, for instance, your university or funding body expects you to have a web presence on their server (e.g., Assyrian Empire Builders [http://www.ucl.ac.uk/sargon/] on the UCL website, as a portal for SAAo [http://oracc.museum.upenn.edu/saao/]).

  4. You can write your own html page(s) following the instructions below, and upload them to your project's 00web directory on Oracc (e.g., DCCMT [http://oracc.museum.upenn.edu/dccmt]). Upload all images for these pages to 00web/images. Ideally your images should be PNG or JPGs. Rebuild your website with the oracc command oracc web. We no longer recommend this option.

  5. If you choose none of the options above, the 40-word blurb and a 320×240 pixel image in your project configuration file will by default serve as your project's front page on the Oracc website (e.g., CAMS [http://oracc.museum.upenn.edu/cams/]).

N.B.: If you choose any of options 1 to 4 you must still complete the image and blurb entries in your configuration file, and also set the public status appropriately in the configuration file; read the section on Listing portal projects for more information.

Creating portal pages for Oracc

If you are creating html pages or whole websites for your Oracc portal, whether on Oracc or on your own computer, you will need to follow these simple guidelines:

  • Your filenames must end with the extension .html, not .htm (e.g., biblio.html).
  • The website's home page must have filename index.html or it will not show up as the homepage.
  • You must link to other web pages, whether on Oracc or elsewhere using full URLs, not the short forms ("tiny URLs") offered by services such as tiny.cc.
  • For links within Oracc follow our citing guidelines, don't just cut and paste from your browser.

Linking to Oracc projects and corpora

Whatever method you use to write portal pages, there is a concise set of recommendations for linking to pages within your corpus and to other Oracc projects. They are set out in a separate document [projurls].

Validating as XHTML Strict

The oracc web and oracc rebuild commands both check that your html pages are valid XHTML Strict and generate error messages if they are not. It is thus important that you validate and clean your html pages with a webservice such as Infohound's HTML Tidy page [http://infohound.net/tidy/] and check and fix links with W3C Link Checker [http://validator.w3.org/checklink].

To validate your HTML or to convert your HTML to XHTML Strict visit the Infohound Tidy webservice [http://infohound.net/tidy/], click on the 'Advanced' selection, and ensure that the following options are selected:

HTML/XHTMLPretty printingEncoding
  • Clean
  • strict Doctype
  • Drop empty paras
  • Numeric Character Refs
  • Output XHTML
  • Word 2000 (if you have
    generated html from
    Word 2000)
  • Numeric entities (this is only visible in Advanced mode, but is very important)
 (leave the defaults
unchanged)
  • UTF8 Char encoding
  • Warnings Output

If you have problems with conversion of characters like em-dashes or smart quotes, try changing the Encoding options--if you are converting an existing web page, you can try looking at the HTML source to see what encoding the document is in. For example, HTML generated by Word on the Mac is likely to be in the 'macintosh' encoding: set the Input Encoding to Mac and the Output Encoding to UTF8 and all will be well. Note that if you are cleaning Word HTML to use in ESP, then if your HTML includes UTF8 characters (as it likely will, for example, š) you will probably need to download the Tidy output then open it in your text editor (e.g., Emacs) and then cut and paste the sections you need into your ESP document--you cannot simply view the Tidy output and cut-paste from that (because Tidy does not include a UTF8 header in its output and the UTF8 characters will be garbled in the Tidy output view).

Note that all text content on Oracc must be in the UTF8 encoding.

We also strongly recommend that you make efforts to ensure your web pages are as accessible as possible to users with disabilities, low-bandwidth internet services, or using mobile devices. The Web Accessibility Initiative's website [http://www.w3.org/WAI/gettingstarted/Overview.html] is a good place to start.

You can address accessibility very easily by using ESP to write your portal pages.

When your portal pages are valid, well linked and accessible, you can upload them to Oracc.

Uploading portal pages to Oracc

If you are creating html pages or whole websites on your own computer to upload to Oracc (option 4 above), you will need to use a Unix terminal such as Putty (PC) or Terminal (Mac).

Before you upload, please check and clean your html pages with a webservice such as HTML Tidy [http://infohound.net/tidy/] and check and fix links with W3C Link Checker. [http://validator.w3.org/checklink]

  • First compress your finished site into a single file (e.g., with File -> Compress in the Finder on a Mac, or with a utility such as 7-Zip [http://www.7-zip.org/] on a PC). Run the zip utility inside the folder with your files in; don't zip the folder itself or you will run into difficulties at the Oracc end.
  • Then upload the ZIP file to your 00web directory on Oracc as described here.
  • Log on to Oracc with a Unix terminal utility and move to your 00web directory by typing cd 00web. Uncompress the ZIP file by typing unzip FILENAME.zip and pressing return. Then delete the ZIP file by typing rm FILENAME.zip and pressing return.
  • Finally, you need to reinstall the new website. Move back to your top level directory by typing cd .. and pressing return. Type oracc web and press return. (For more information, see the page on The Oracc Command.) Log out of Oracc.

If the oracc web command takes an inordinately long time to execute, edit the <!DOCTYPE /> element in your html files to reference /xhtml-strict.dtd instead of http://www.w3.org/xhtml-strict.dtd - ask your liaison if you need help with this.

Linking portal and corpus

For both options 1 and 2 above, you will need to edit the pager-back-button and pager-back-url options in your corpus project's configuration file in order to link the two together.

Listing portal projects

Portal projects that are hosted on Oracc (i.e., option 3 above) are included in the public Project List [http://oracc.museum.upenn.edu/project-list.html] by default. You can set the blurb and image in that list by editing the your project's configuration file.

If you would like your non-Oracc portal site (i.e., option 2 above) to be included in the Project List [http://oracc.museum.upenn.edu/project-list.html], then you need to make a configuration file for it, following this model:

<project xmlns="http://oracc.org/ns/xpd/1.0" 
	 xmlns:xpd="http://oracc.org/ns/xpd/1.0"
	 n="knp">
  <name>Knowledge and Power in the Neo-Assyrian Empire</name>
  <abbrev>K&P</abbrev>
  <url>http://knp.prs.heacademy.ac.uk</url>
  <public>yes</public>
  <blurb>This website presents Neo-Assyrian scholars' letters, queries, and 
  reports to their kings in seventh-century Nineveh and provides resources 
  to support their use in undergraduate teaching.</blurb>
  <image>palace.jpg</image>
  <image-alt>An Assyrian king with his scribes and scholars, as imagined 
  in the mid-19th century. (A.H. Layard, A Second Series of the Monuments 
  of Nineveh, London 1853, pl. 2 detail, after a sketch by J. 
  Fergusson).</image-alt>
</project>

Copy this text, edit it in Emacs to suit your portal, and save the file as config.xml. Email it, together with a suitable 320×240 pixel image, to your project liaison for him or her to install it in the projects listing directory on Oracc.
